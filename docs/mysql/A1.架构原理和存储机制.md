3. 为什么 innodb 必须建主键？

   如果不建主键，mysql 会找到带 union index 的列作为主键列，如果找不到，默认创建隐藏列 row Id 作为主键

4. 为什么用自增的整型而不是 uuid 作为主键？

   - 空间小
   - uuid 大小比较比整型慢
   - 趋势自增 -> 自增
   - 递增的索引，基本都是从后面去插入新的叶子节点，很少会发生节点分裂，维护索引的效率更高
